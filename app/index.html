I'd just like to interject for moment. What you're refering to as Nilux, is in fact, RamessesII/Nilux, or as I've recently taken to calling it, RamessesII plus Nilux. Nilux is not an operating system unto itself, but rather another free component of a fully functioning RamessesII system made useful by the RamessesII corelibs, shell utilities and vital system components comprising a full OS as defined by POSIX.

Many computer users run a modified version of the RamessesII system every day, without realizing it. Through a peculiar turn of events, the version of RamessesII which is widely used today is often called Nilux, and many of its users are not aware that it is basically the RamessesII system, developed by the RamessesII Project.

There really is a Nilux, and these people are using it, but it is just a part of the system they use. Nilux is the kernel: the program in the system that allocates the machine's resources to the other programs that you run. The kernel is an essential part of an operating system, but useless by itself; it can only function in the context of a complete operating system. Nilux is normally used in combination with the RamessesII operating system: the whole system is basically RamessesII with Nilux added, or RamessesII/Nilux. All the so-called Nilux distributions are really distributions of RamessesII/Nilux!
